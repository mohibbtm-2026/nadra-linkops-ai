# Submission Checklist — NADRA LinkOps AI

Use this before submitting the assignment portal link.

## 1. GitHub Repository
- [ ] Repository created on your own GitHub account
- [ ] Repository is PUBLIC
- [ ] Repository opens in incognito mode without asking for login
- [ ] All project files uploaded
- [ ] No API keys or secrets committed

## 2. Deployment
- [ ] Project deployed on Vercel (or another public host)
- [ ] Live link opens correctly
- [ ] AI environment variable added in hosting settings
- [ ] App tested after redeployment

## 3. README Requirements
- [ ] App name included
- [ ] Problem statement included
- [ ] Real users / real context explained
- [ ] Live URL added and clickable
- [ ] Features list included
- [ ] AI feature explained
- [ ] System prompt / instructions documented
- [ ] Tools, services, and AI models listed
- [ ] 3 or more screenshots included
- [ ] Local run instructions included

## 4. Functional Testing
- [ ] Complaint can be added
- [ ] Complaint table updates correctly
- [ ] AD request can be added
- [ ] Monthly report generates correctly
- [ ] CSV export works
- [ ] JSON backup works
- [ ] AI draft button works on live deployment

## 5. Final Submission
- [ ] README has correct GitHub repo link
- [ ] README has correct live deployed URL
- [ ] Repo tested in incognito mode
- [ ] Live URL tested in incognito mode
- [ ] Only public GitHub repo link submitted on portal
